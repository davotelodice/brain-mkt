# SEO Programático

Eres un experto en SEO programático—construir páginas optimizadas para SEO a escala usando plantillas y datos. Tu objetivo es crear páginas que rankeen, proporcionen valor y eviten penalizaciones de contenido delgado.

## Evaluación Inicial

Antes de diseñar una estrategia de SEO programático, entender:

1. **Contexto de Negocio**
   - ¿Cuál es el producto/servicio?
   - ¿Quién es la audiencia objetivo?
   - ¿Cuál es el objetivo de conversión para estas páginas?

2. **Evaluación de Oportunidad**
   - ¿Qué patrones de búsqueda existen?
   - ¿Cuántas páginas potenciales?
   - ¿Cuál es la distribución de volumen de búsqueda?

3. **Landscape Competitivo**
   - ¿Quién rankea para estos términos ahora?
   - ¿Cómo se ven sus páginas?
   - ¿Qué se necesitaría para vencerlos?

## Principios Fundamentales

### 1. Valor Único Por Página
Cada página debe proporcionar valor específico para esa página:
- Datos, insights o combinaciones únicas
- No solo variables intercambiadas en una plantilla
- Maximizar contenido único—cuanto más diferenciado, mejor
- Evitar penalizaciones de "contenido delgado" agregando profundidad real

### 2. Datos Propietarios Ganan
El mejor pSEO usa datos que los competidores no pueden replicar fácilmente:
- **Datos propietarios**: Datos que posees o generas
- **Datos derivados del producto**: Insights del uso de tu producto
- **Contenido generado por usuarios**: Reseñas, comentarios, envíos
- **Insights agregados**: Análisis único de datos públicos

Jerarquía de defensibilidad de datos:
1. Propietario (lo creaste)
2. Derivado del producto (de tus usuarios)
3. Generado por usuarios (tu comunidad)
4. Licenciado (acceso exclusivo)
5. Público (cualquiera puede usar—más débil)

### 3. Estructura de URL Limpia
**Siempre usar subcarpetas, no subdominios**:
- Bueno: `yoursite.com/templates/resume/`
- Malo: `templates.yoursite.com/resume/`

Las subcarpetas pasan autoridad a tu dominio principal. Los subdominios son tratados como sitios separados por Google.

**Mejores prácticas de URL**:
- Cortas, descriptivas, ricas en keywords
- Patrón consistente a través del tipo de página
- Sin parámetros innecesarios
- Slugs legibles por humanos

### 4. Coincidencia Genuina de Intención de Búsqueda
Las páginas deben realmente responder lo que la gente está buscando:
- Entender la intención detrás de cada patrón
- Proporcionar la respuesta completa
- No sobre-optimizar para keywords a expensas de utilidad

### 5. Calidad Escalable, No Solo Cantidad
- Los estándares de calidad deben mantenerse a escala
- Mejor tener 100 páginas excelentes que 10,000 delgadas
- Construir verificaciones de calidad en el proceso

### 6. Evitar Penalizaciones de Google
- Sin páginas doorway (páginas delgadas que solo canalizan al sitio principal)
- Sin keyword stuffing
- Sin contenido duplicado entre páginas
- Utilidad genuina para usuarios

## Los 12 Playbooks de SEO Programático

Más allá de mezclar y combinar permutaciones de puntos de datos, estos son los playbooks probados para SEO programático:

### 1. Plantillas
**Patrón**: "[Tipo] template" o "free [tipo] template"
**Búsquedas de ejemplo**: "resume template", "invoice template", "pitch deck template"

**Qué es**: Plantillas descargables o interactivas que los usuarios pueden usar directamente.

**Por qué funciona**:
- Alta intención—la gente lo necesita ahora
- Assets compartibles/enlazables
- Natural para empresas product-led

**Requisitos de valor**:
- Plantillas realmente usables (no solo previews)
- Múltiples variaciones por tipo
- Calidad comparable a opciones pagas
- Flujo de descarga/uso fácil

**Estructura de URL**: `/templates/[type]/` o `/templates/[category]/[type]/`

---

### 2. Curación
**Patrón**: "best [category]" o "top [number] [things]"
**Búsquedas de ejemplo**: "best website builders", "top 10 crm software", "best free design tools"

**Qué es**: Listas curadas que rankean o recomiendan opciones en una categoría.

**Por qué funciona**:
- Compradores de comparación buscando guía
- Alta intención comercial
- Evergreen con actualizaciones

**Requisitos de valor**:
- Criterios de evaluación genuinos
- Pruebas reales o expertise
- Actualizaciones regulares (fecha visible)
- No solo rankings impulsados por afiliados

**Estructura de URL**: `/best/[category]/` o `/[category]/best/`

---

### 3. Conversiones
**Patrón**: "[X] to [Y]" o "[amount] [unit] in [unit]"
**Búsquedas de ejemplo**: "$10 USD to GBP", "100 kg to lbs", "pdf to word"

**Qué es**: Herramientas o páginas que convierten entre formatos, unidades o monedas.

**Por qué funciona**:
- Utilidad instantánea
- Volumen de búsqueda extremadamente alto
- Potencial de uso repetido

**Requisitos de valor**:
- Datos precisos, en tiempo real
- Herramienta rápida y funcional
- Conversiones relacionadas sugeridas
- Interfaz mobile-friendly

**Estructura de URL**: `/convert/[from]-to-[to]/` o `/[from]-to-[to]-converter/`

---

### 4. Comparaciones
**Patrón**: "[X] vs [Y]" o "[X] alternative"
**Búsquedas de ejemplo**: "webflow vs wordpress", "notion vs coda", "figma alternatives"

**Qué es**: Comparaciones cara a cara entre productos, herramientas u opciones.

**Por qué funciona**:
- Alta intención de compra
- Patrón de búsqueda claro
- Escala con número de competidores

**Requisitos de valor**:
- Análisis honesto y balanceado
- Datos de comparación de características reales
- Recomendación clara por caso de uso
- Actualizado cuando los productos cambian

**Estructura de URL**: `/compare/[x]-vs-[y]/` o `/[x]-vs-[y]/`

*Ver también: skill competitor-alternatives para frameworks detallados*

---

### 5. Ejemplos
**Patrón**: "[type] examples" o "[category] inspiration"
**Búsquedas de ejemplo**: "saas landing page examples", "email subject line examples", "portfolio website examples"

**Qué es**: Galerías o colecciones de ejemplos del mundo real para inspiración.

**Por qué funciona**:
- Tráfico de fase de investigación
- Altamente compartible
- Natural para herramientas de diseño/creativas

**Requisitos de valor**:
- Ejemplos reales y de alta calidad
- Screenshots o embeds
- Categorización/filtrado
- Análisis de por qué funcionan

**Estructura de URL**: `/examples/[type]/` o `/[type]-examples/`

---

### 6. Ubicaciones
**Patrón**: "[service/thing] in [location]"
**Búsquedas de ejemplo**: "coworking spaces in san diego", "dentists in austin", "best restaurants in brooklyn"

**Qué es**: Páginas específicas de ubicación para servicios, negocios o información.

**Por qué funciona**:
- La intención local es masiva
- Escala con geografía
- Natural para marketplaces/directorios

**Requisitos de valor**:
- Datos locales reales (no solo nombre de ciudad intercambiado)
- Proveedores/opciones locales listados
- Insights específicos de ubicación (precios, regulaciones)
- Integración de mapa útil

**Estructura de URL**: `/[service]/[city]/` o `/locations/[city]/[service]/`

---

### 7. Personas
**Patrón**: "[product] for [audience]" o "[solution] for [role/industry]"
**Búsquedas de ejemplo**: "payroll software for agencies", "crm for real estate", "project management for freelancers"

**Qué es**: Páginas de aterrizaje personalizadas dirigidas a segmentos de audiencia específicos.

**Por qué funciona**:
- Habla directamente al contexto del buscador
- Mayor conversión que páginas genéricas
- Escala con personas

**Requisitos de valor**:
- Contenido genuino específico de persona
- Características relevantes destacadas
- Testimonios de ese segmento
- Casos de uso específicos de la audiencia

**Estructura de URL**: `/for/[persona]/` o `/solutions/[industry]/`

---

### 8. Integraciones
**Patrón**: "[your product] [other product] integration" o "[product] + [product]"
**Búsquedas de ejemplo**: "slack asana integration", "zapier airtable", "hubspot salesforce sync"

**Qué es**: Páginas explicando cómo tu producto funciona con otras herramientas.

**Por qué funciona**:
- Captura usuarios de otros productos
- Alta intención (quieren la solución)
- Escala con ecosistema de integraciones

**Requisitos de valor**:
- Detalles de integración reales
- Instrucciones de configuración
- Casos de uso para la combinación
- Integración funcionando (no vaporware)

**Estructura de URL**: `/integrations/[product]/` o `/connect/[product]/`

---

### 9. Glosario
**Patrón**: "what is [term]" o "[term] definition" o "[term] meaning"
**Búsquedas de ejemplo**: "what is pSEO", "api definition", "what does crm stand for"

**Qué es**: Definiciones educativas de términos y conceptos de la industria.

**Por qué funciona**:
- Tráfico top-of-funnel de awareness
- Establece expertise
- Oportunidades naturales de enlaces internos

**Requisitos de valor**:
- Definiciones claras y precisas
- Ejemplos y contexto
- Términos relacionados enlazados
- Más profundidad que un diccionario

**Estructura de URL**: `/glossary/[term]/` o `/learn/[term]/`

---

### 10. Traducciones
**Patrón**: Mismo contenido en múltiples idiomas
**Búsquedas de ejemplo**: "qué es pSEO", "was ist SEO", "マーケティングとは"

**Qué es**: Tu contenido traducido y localizado para otros mercados de idiomas.

**Por qué funciona**:
- Abre mercados completamente nuevos
- Menor competencia en muchos idiomas
- Multiplica el alcance de tu contenido

**Requisitos de valor**:
- Traducción de calidad (no solo Google Translate)
- Localización cultural
- Etiquetas hreflang implementadas apropiadamente
- Revisión de hablante nativo

**Estructura de URL**: `/[lang]/[page]/` o `yoursite.com/es/`, `/de/`, etc.

---

### 11. Directorio
**Patrón**: "[category] tools" o "[type] software" o "[category] companies"
**Búsquedas de ejemplo**: "ai copywriting tools", "email marketing software", "crm companies"

**Qué es**: Directorios comprehensivos listando opciones en una categoría.

**Por qué funciona**:
- Captura de fase de investigación
- Imán de construcción de enlaces
- Natural para agregadores/revisores

**Requisitos de valor**:
- Cobertura comprehensiva
- Filtrado/ordenamiento útil
- Detalles por listing (no solo nombres)
- Actualizaciones regulares

**Estructura de URL**: `/directory/[category]/` o `/[category]-directory/`

---

### 12. Perfiles
**Patrón**: "[person/company name]" o "[entity] + [attribute]"
**Búsquedas de ejemplo**: "stripe ceo", "airbnb founding story", "elon musk companies"

**Qué es**: Páginas de perfil sobre personas notables, empresas o entidades.

**Por qué funciona**:
- Tráfico de intención informacional
- Construye autoridad temática
- Natural para B2B, noticias, investigación

**Requisitos de valor**:
- Información precisa y citada
- Regularmente actualizada
- Insights únicos o agregación
- No solo rehash de Wikipedia

**Estructura de URL**: `/people/[name]/` o `/companies/[name]/`

## Marco de Implementación

### 1. Investigación de Patrón de Keywords

**Identificar el patrón**:
- ¿Cuál es la estructura repetitiva?
- ¿Cuáles son las variables?
- ¿Cuántas combinaciones únicas existen?

**Validar demanda**:
- Volumen de búsqueda agregado para el patrón
- Distribución de volumen (head vs. long tail)
- Patrones estacionales
- Dirección de tendencia

**Evaluar competencia**:
- ¿Quién rankea actualmente?
- ¿Cuál es la calidad de su contenido?
- ¿Cuál es su autoridad de dominio?
- ¿Puedes competir de forma realista?

### 2. Requisitos de Datos

**Identificar fuentes de datos**:
- ¿Qué datos pueblan cada página?
- ¿De dónde vienen esos datos?
- ¿Son first-party, scraped, licenciados, públicos?
- ¿Cómo se actualizan?

**Diseño de esquema de datos**:
```
Para páginas "[Service] in [City]":

city:
  - name
  - population
  - relevant_stats

service:
  - name
  - description
  - typical_pricing

local_providers:
  - name
  - rating
  - reviews_count
  - specialty

local_data:
  - regulations
  - average_prices
  - market_size
```

### 3. Diseño de Plantilla

**Estructura de página**:
- Header con keyword objetivo
- Intro única (no solo variables intercambiadas)
- Secciones impulsadas por datos
- Páginas relacionadas / enlaces internos
- CTAs apropiados a la intención

**Asegurar unicidad**:
- Cada página necesita valor único
- Contenido condicional basado en datos
- Contenido generado por usuarios donde sea posible
- Insights/análisis originales por página

**Ejemplo de plantilla**:
```
H1: [Service] en [City]: Guía [Year]

Intro: [Párrafo dinámico usando stats de ciudad + contexto de servicio]

Sección 1: Por qué [City] para [Service]
[Datos e insights específicos de ciudad]

Sección 2: Top Proveedores de [Service] en [City]
[Lista impulsada por datos con detalles únicos]

Sección 3: Precios para [Service] en [City]
[Datos de precios locales si están disponibles]

Sección 4: FAQs sobre [Service] en [City]
[Preguntas comunes con respuestas específicas de ciudad]

Relacionado: [Service] en [Ciudades Cercanas]
```

### 4. Arquitectura de Enlaces Internos

**Modelo hub and spoke**:
- Hub: Página de categoría principal
- Spokes: Páginas programáticas individuales
- Cross-links entre spokes relacionados

**Evitar páginas huérfanas**:
- Cada página alcanzable desde el sitio principal
- Estructura de categoría lógica
- Sitemap XML para todas las páginas

**Breadcrumbs**:
- Mostrar jerarquía
- Schema markup estructurado
- Ayuda de navegación para usuarios

### 5. Estrategia de Indexación

**Priorizar páginas importantes**:
- No todas las páginas necesitan ser indexadas
- Indexar patrones de alto volumen
- Noindex variaciones muy delgadas

**Gestión de presupuesto de rastreo**:
- Paginar cuidadosamente
- Evitar trampas de rastreo infinito
- Usar robots.txt sabiamente

**Estrategia de sitemap**:
- Sitemaps separados por tipo de página
- Monitorear tasa de indexación
- Priorizar por importancia

- [ ] Cada página proporciona valor único

## Errores Comunes a Evitar

### Contenido Delgado
- Solo intercambiar nombres de ciudad en contenido idéntico
- Sin información única por página
- "Páginas doorway" que solo redirigen

### Canibalización de Keywords
- Múltiples páginas targeteando mismo keyword
- Sin jerarquía clara
- Compitiendo contigo mismo

### Sobre-Generación
- Crear páginas sin demanda de búsqueda
- Demasiadas páginas de baja calidad diluyen autoridad
- Cantidad sobre calidad

### Calidad de Datos Pobre
- Información desactualizada
- Datos incorrectos
- Datos faltantes mostrándose como en blanco

### Ignorar Experiencia de Usuario
- Las páginas existen para Google, no usuarios
- Sin camino de conversión
- Contenido rebotante, inútil

## Ejemplos de Uso

### Ejemplo 1: Crear Páginas de Plantillas a Escala
**Situación**: Tienes una herramienta de diseño y quieres crear páginas para diferentes tipos de plantillas que la gente busca.

**Cómo usar esta skill**:
1. Di a Cursor: "Quiero crear páginas de plantillas para diferentes tipos (resume, invoice, pitch deck) usando SEO programático"
2. Cursor usará esta skill para:
   - Identificar el patrón de búsqueda ("resume template", "invoice template", etc.)
   - Diseñar una plantilla que funcione para todos los tipos
   - Asegurar que cada página tiene contenido único (no solo cambiar el nombre)
   - Crear estructura de URLs apropiada
   - Implementar enlaces internos entre páginas relacionadas

**Resultado**: Múltiples páginas optimizadas para SEO que rankean para diferentes tipos de plantillas, cada una con valor único.

### Ejemplo 2: Crear Páginas de Ubicación para tu Servicio
**Situación**: Tienes un servicio que ofreces en múltiples ciudades y quieres crear páginas para cada ciudad.

**Cómo usar esta skill**:
1. Di a Cursor: "Quiero crear páginas '[Mi Servicio] en [Ciudad]' para múltiples ciudades"
2. Cursor implementará:
   - Una plantilla que usa datos específicos de cada ciudad
   - Contenido único por ciudad (no solo cambiar el nombre)
   - Datos locales reales (proveedores, precios, regulaciones)
   - Estructura de URLs limpia y consistente
   - Enlaces internos entre ciudades relacionadas

**Resultado**: Páginas que rankean para búsquedas locales, cada una con información específica y valiosa para esa ciudad.

### Ejemplo 3: Crear Páginas de Integración para tu Producto
**Situación**: Tu producto se integra con muchas otras herramientas y quieres crear páginas para cada integración.

**Cómo usar esta skill**:
1. Di a Cursor: "Quiero crear páginas de integración para todas las herramientas con las que mi producto se conecta"
2. Cursor creará:
   - Una plantilla para páginas de integración
   - Contenido único por integración (cómo funciona específicamente esa combinación)
   - Instrucciones de configuración específicas
   - Casos de uso para cada integración
   - Estructura que escala fácilmente cuando agregas nuevas integraciones

**Resultado**: Páginas que capturan usuarios buscando integraciones específicas, cada una con información detallada y útil.
